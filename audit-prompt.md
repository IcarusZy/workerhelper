请审查当前 WorkerHelper 技能组，但只输出“必须整改”的问题。

审查目标仅限以下两项：

1. 是否支撑设计目标：
   - 通过功能路由图快速定位功能点对应的核心代码
   - 规范编码执行流程，并按任务规模进入 worker-lookup / design / plan / exec-plan / worker-sync
2. 各技能的 description 与正文是否存在会影响使用的严重问题：
   - 触发条件不清
   - 技能边界冲突
   - 工作流互相矛盾
   - 关键职责缺失
   - 描述重复到可能误导模型
   - 输出要求不一致导致后续技能无法接续

不要输出以下内容：

- 纯措辞优化
- 风格统一建议
- 不影响执行的重复表达
- 可选增强项
- 个人偏好的结构调整
- 没有明确证据的推测

每个问题必须同时满足：

- 能指出具体文件和行号或段落
- 能说明它如何破坏上述两个设计目标之一
- 能给出最小整改方向

输出格式：

## 必须整改项

- 文件：`path`
- 位置：...
- 问题：...
- 影响：...
- 最小整改方向：...

如果没有满足以上标准的问题，只输出：

无必须整改项。
